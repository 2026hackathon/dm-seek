1. 一句话结论

Delivery RCA 是一条跨服务异步流水线：用户在 BO 后台提交 RCA → delivery-task-v2-service 聚合 8+ 数据源构建请求载荷 → 通过 REST 同步调用 delivery-agent-service → Google ADK LLM Agent（Gemini）并行执行 RCA 分析 + Delivery 分析 → 结果通过 Kafka 异步回传给 task-v2 → 写入 MongoDB，前端轮询获取。

核心 2 个 LLM Agent（DeliveryRcaAgent + DeliveryAnalysisAgent）在 DeliveryRcaRunnerService 中通过 RxJava Flowable.merge() 实现真正并行执行，结果经栅栏检查后聚合发布。

2. 架构总览

Agent 框架Google ADK (com.google.adk)，InMemoryRunner
LLM 模型Gemini（可配置：DeliveryRcaAgentSchema.RCA_AGENT_MODEL）
并行策略RxJava Flowable.merge() 双 Agent 并行 + 栅栏聚合
持久化MongoDB（JobRcaRequestHistory + JobRcaAnalyticsLog）
入口点类型REST WebService + Kafka MessageHandler

6.4 结果聚合 — 栅栏机制

tryCompleteAndPublish() 检查：
  1. RCA Agent 是否完成?  (IS_RCA_FINISHED)
  2. Analysis Agent 是否完成? (若需要: IS_ANALYSIS_FINISHED)
  3. 是否已发布过?  (RCA_RESULT_PUBLISHED 防重)
  → 全部满足 → 聚合 usage metadata → 构建 DeliveryRcaOutputSchema → publishEvent()
