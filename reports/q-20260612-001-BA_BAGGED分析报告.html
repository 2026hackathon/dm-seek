📊 最终结果

🔴 根因：DELI-4475 中期 Scope Expansion

C1 — 业务根因

DELI-4475（"Wonder Local and 3P Legacy Cleanup"）在实现中期（2026-05-14）将清理范围从 "Wonder Local only" 扩大到 "Wonder Local + 3P"。扩大后直接删除了 MongoSourceType.isHDRSourceType() 方法（原语义：return this != WONDER_LOCAL），导致 buildDefaultAllocationSetting() 中的方法引用编译失败。开发者被迫替换为硬编码 filter（仅 APP + CORPORATE），BA_SHIPMENT_BAGGED 和 BA_SHIPMENT_BOXED 被静默排除。

HDRDeliverySettingBuilder.java buildDefaultAllocationSetting() commit 91d86ea28 DELI-4475

C2 — 技术根因

isHDRSourceType() 方法被直接删除而非标记 @Deprecated 或逐一重构调用方。该方法的语义是「排除 Wonder Local」→ 隐含保证了 BA 类型被包含。删除时未检查调用方对枚举值覆盖范围的隐含假设。

// HDRDeliverySettingBuilder.java — buildDefaultAllocationSetting()

// DELI-4475 之前（正常）:
Arrays.stream(MongoSourceType.values())
    .filter(MongoSourceType::isHDRSourceType)  // isHDRSourceType(): this != WONDER_LOCAL
    .map(...)
    // → 生成 APP + THIRD_PARTY + BA_BOXED + BA_BAGGED + CORPORATE 共 5 个 ✅

// DELI-4475 之后（缺陷）:
Arrays.stream(MongoSourceType.values())
    .filter(s -> s == MongoSourceType.APP || s == MongoSourceType.CORPORATE)
    .map(...)
    // → 只生成 APP + CORPORATE 共 2 个 ❌ BA_BAGGED 消失

// DELI-4520 修复（当前）:
Arrays.stream(MongoSourceType.values())
    .map(HDRDeliverySettingBuilder::buildDefaultAllocationSetting)
    .toList()
    // → 遍历全部 4 个枚举值 ✅ 免疫未来变化
