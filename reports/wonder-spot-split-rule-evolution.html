DELI-2847 · Wonder Spot Delivery Creation — 原始创建

Wonder Spot 配送系统的诞生。拆单规则从第一天就刻在基因里：

  • 分组键：CorporateTaskGroupKey(corporateId, groupNumber) — 与现在完全相同

  • tryAppend：尝试追加到已有 Job 的框架已存在
  • APPENDABLE_JOB_STATUSES：{CREATED, ASSIGNED, PICKUP_ENROUTE, PICKUP_ARRIVED}
  • 原始 Item 上限：wonderSpotMaxOrderItemCount（全局配置）

2025-03-10 ~ 2025-03-17
Phase 2
DELI-3406 → DELI-3408 · 生产事故驱动修复

DELI-3406（生产事故）：Wonder Spot Delivery 的 Ready Event 未被触发，订单卡住无法进入取餐流程。

DELI-3408（修复）：

  • 忽略 15 分钟限制：provider 分配失败时不阻塞 Job 创建
  • job_id 全链路迁移：provider 失败重试时确保旧 job_id 正确迁移到新 Job
2025-03-28 ~ 2025-04-03
Phase 3
DELI-3472 · 乐观锁 + 独立 Consumer

父需求 DELI-3447（并发安全）：

  • 乐观锁：jobRepository.partialUpdate 带版本号条件，防止并发 Append 时 data race 
    success = jobRepository.partialUpdate(appendedJob.job,
    "job_status IN (...) AND version = ?",  // ← 乐观锁
    APPENDABLE_JOB_STATUSES, currentVersion);
  • 独立 consumer group：Kafka consumer 隔离，避免与其他服务竞争
2025-04-24 ~ 2025-04-27
Phase 4
DELI-3614 · Append 三级规则 — 最大飞跃

父需求 DELI-2476（拆单/合批总纲）。三个关键变更：

  • 4a. 新增 APPENDABLE_JOB_ORDER_STATUSES：在 JobStatus 过滤之上增加 OrderStatus 维度
    OrderStatus追加？理由
    NEW, ACCEPTED, HELD, ASSIGNED_TO_POD, IN_COOKING✅备货阶段
    PENDING_PACKAGE, PENDING_BAGGING, READY_FOR_PICKUP❌已进入出货流程
    CANCELED❌已取消
  • 4b. Pre-allocate 机制：新增 appendToPreAllocatedJobs，Append 从单级升级为两级
  • 4c. 15min Release Buffer：PRE_ALLOCATE_CORPORATE_BUFFER_IN_MINUETS = 15
  • 4d. mergeJobOrderStatusTracking：Job 创建后合并 Task 级别 tracking
2025-04-27
Phase 5
DELI-3632 · Cancel 边界修复
  • 场景：预分配 Job 中所有 task 被 cancel → 后续同时间窗新 task 无法创建
  • 修复：isPreAllocatedJobAppendable 增加全 cancel 检查，全 cancel 的预分配 Job 被过滤，允许新建
2026-05-28 ~ 2026-05-29
Phase 6
DELI-4494 · GH Wonder Spot 30min 缓冲
  • GH Wonder Spot group order 的预分配 release buffer 15 → 30 分钟
  • 通过 DevCycle feature flag getWonderSpotGH30MinReleaseTargetCorporates 灰度控制
  • private int resolveReleaseBufferMinutes(DBProviderType providerType, String corporateId) {
    if (providerType == GRUBHUB
        && featureFlagService.getWonderSpotGH30MinReleaseTargetCorporates().contains(corporateId)) {
        return 30;  // GH Wonder Spot 特定 corporate
    }
    return 15;      // 默认
}

// CorporateJobCreationService.java — 拆单规则完整流程

// 1. 分组键（从 DELI-2847 延续至今）
class CorporateTaskGroupKey {
    String corporateId;
    String groupNumber;  // = "{deliveryTimeslotStart}_{deliveryTimeslotEnd}"
}

// 2. 可追加条件（JobStatus + OrderStatus 双维度）
static final APPENDABLE_JOB_STATUSES = {
    CREATED, ASSIGNED, PICKUP_ENROUTE, PICKUP_ARRIVED
};
static final APPENDABLE_JOB_ORDER_STATUSES = {
    NEW, ACCEPTED, HELD, ASSIGNED_TO_POD, IN_COOKING
};

// 3. 两级 Append
List tryAppend(corporateId, groupNumber, taskParams, setting) {
    // ① 先试 Pre-allocated Jobs
    result = appendToPreAllocatedJobs(corporateId, groupNumber, taskParams);
    if (!result.isEmpty()) return result;
    // ② 再试 Truly Created Jobs
    return appendToTrulyCreationJobs(corporateId, groupNumber, taskParams);
}

// 4. Release Buffer（DELI-4494 定制）
int resolveReleaseBufferMinutes(providerType, corporateId) {
    if (providerType == GRUBHUB
        && featureFlagService.getWonderSpotGH30MinReleaseTargetCorporates()
            .contains(corporateId)) {
        return 30;  // GH Wonder Spot 特定
    }
    return 15;      // 默认
}

// 5. 乐观锁（DELI-3472）
success = jobRepository.partialUpdate(job,
    "job_status IN (...) AND version = ?",
    APPENDABLE_JOB_STATUSES, currentVersion);
