📌 执行摘要

allocation-setting 是 HDRDeliverySetting 中按订单来源类型（MongoSourceType）分别配置的配送商分配设置。每个来源类型（APP、CORPORATE、BA_SHIPMENT_BAGGED、BA_SHIPMENT_BOXED）各有一条 allocation setting，控制该类型订单走哪些配送商、是自动派单还是运营手动推。

1.2 当前实现（DELI-4520 修复后）

// HDRDeliverySettingBuilder.java L98-107
public static List<HDRAllocationSetting> buildDefaultAllocationSetting() {
    return Arrays.stream(MongoSourceType.values())
        .map(HDRDeliverySettingBuilder::buildDefaultAllocationSetting)
        .toList();
}

public static HDRAllocationSetting buildDefaultAllocationSetting(MongoSourceType sourceType) {
    var setting = new HDRAllocationSetting();
    setting.sourceType = sourceType;
    setting.providerTypes = DEFAULT_ALLOCATION_PROVIDERS;  // [RELAY, GRUBHUB, DOORDASH, NASH]
    return setting;
}

1.4 历史 Bug 背景（DELI-4475 → DELI-4520）

DELI-4475 缺陷版本（2026-05-14 ~ 2026-06-11）—— 把 .filter(MongoSourceType::isHDRSourceType) 替换为硬编码 .filter(s -> s == APP || s == CORPORATE)，导致 BA_BAGGED 和 BA_BOXED 的 allocation setting 静默消失。

DELI-4520 修复（2026-06-11）—— 改为 .map(buildDefaultAllocationSetting) 遍历全部枚举值，消除硬编码 filter，对未来的枚举增减具有免疫力。
